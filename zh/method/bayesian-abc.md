# 2. 贝叶斯冲突预测（ABC）

核心预测引擎使用**近似贝叶斯计算**（ABC），通过50,000次蒙特卡罗模拟来估算四种冲突情景的后验概率。

## 为什么使用ABC？

传统贝叶斯推断需要似然函数，但对于复杂的冲突动态来说这是不可计算的。ABC通过以下方式绕过这一问题：
1. 从每种情景的生成模型中模拟攻击轨迹
2. 使用距离度量将模拟轨迹与观测数据进行比较
3. 根据模拟与现实的匹配程度对情景进行加权

## 情景定义

| 情景 | β衰减率 | 无人机率 | 巡航导弹率 | 持续时间 | 升级概率 |
|------|---------|---------|-----------|---------|---------|
| **停火** | 0.50 | 60架/天 | 0.3枚/天 | 7 ± 3天 | 0% |
| **基线** | 0.35 | 130架/天 | 1.0枚/天 | 20 ± 6天 | 3% |
| **升级** | 0.18 | 180架/天 | 2.5枚/天 | 35 ± 10天 | 10% |
| **全面战争** | 0.08 | 300架/天 | 6.0枚/天 | 60 ± 15天 | 25% |

每种情景由 `ScenarioParams` 数据类定义，参数涵盖弹道导弹衰减率、无人机产能、巡航导弹频率、预期冲突持续时间和升级概率。

## 软ABC算法

与使用硬拒绝阈值的标准ABC不同，本实现使用**带高斯核加权的软ABC**。所有模拟均被保留，但按拟合质量加权：

```
NRMSE_i = RMSE(sim_i[0:T_obs], observed) / mean(observed)

w_i = exp(-0.5 · (NRMSE_i / h)²) · π_s
```

其中：
- `h = 0.35` 为核带宽
- `π_s` 为情景s的信息先验

### 信息先验

```python
prior_weights = {
    "ceasefire": 0.05,     # 低先验 — 尚无停火信号
    "baseline": 0.45,      # 最高先验 — 与数据最一致
    "escalation": 0.35,    # 较高先验 — 尾部风险
    "regime_war": 0.15     # 小但非零 — 极端情景
}
```

## 时变后验演化

一个关键创新：后验概率在观测窗口之外**随时间演化**。这反映了物理现实——被摧毁的TEL（运输-起竖-发射车）无法补充，使得持续弹道攻击行动的可行性逐渐降低。

### 增强函数

```
boost_ceasefire(d)   = 1 + 3.0 · (1 - exp(-0.15 · (d - T_obs)))
boost_baseline(d)    = 1 + 0.5 · exp(-0.05 · (d - T_obs))
boost_escalation(d)  = max(0.1, 1 - 0.7 · (1 - exp(-0.10 · (d - T_obs))))
boost_regime_war(d)  = max(0.05, 1 - 0.85 · (1 - exp(-0.08 · (d - T_obs))))
```

**解读：**
- 停火概率随时间**增长**，因为发射装置消耗持续累积
- 基线保持相对稳定，初期略有上升
- 升级和全面战争概率**衰减**，因为物理升级能力逐渐减弱

## 输出

ABC模块产出：
- 每天每种情景的**后验概率**（30天展望）
- 按情景的**攻击预测**，含中位数和50%/90%置信区间
- **有效样本量（ESS）**，用于评估后验质量
- 第6、14、30天里程碑处的**概率摘要**
